# AGENTS

- Stack: Laravel 12 + Vue 3/Vite + Vuetify; domain modules live under `Modules/{Auth,CRM,Chat,HRM}` and are wired in `bootstrap/providers.php`.
- DI & repos: Module service providers bind interfaces to repositories (e.g., `Modules/HRM/Providers/HRMServiceProvider.php` binds `EmployeeRepositoryInterface` -> `EmployeeRepository`; CRM provider binds product/report/notice/post repos). Follow this binding pattern for new contracts.
- Module loading: Providers also `mergeConfigFrom`, `loadMigrationsFrom`, and `loadViewsFrom` their module paths; keep new configs/migrations inside the module and register via the provider.
- Controllers: Prefer repository injection + request validators/resources (see `Modules/HRM/Http/Controllers/EmployeeController.php` using `EmployeeRequest` + `EmployeeResource`). Many endpoints stream Excel/PDF via `maatwebsite/excel` and `barryvdh/laravel-dompdf`—reuse those helpers for exports.
- Queues/jobs: Attendance automation lives in `app/Jobs/*` (e.g., `AddAttendanceRecords`, `AutomateAttendanceJob`, `UpdateAttendanceStatus`); dev script runs a queue listener—ensure new jobs are queued, not blocking. There is a dedicated `crm` queue connection in `config/queue.php` for CRM-related jobs, in addition to the default `database` queue.
- Permissions: Spatie roles/permissions; backend gates routes with `can:*`. Frontend permission cache is `localStorage.loggedInUserPermission` + `useCookie('userData')` and helpers in `resources/js/utils/permission.js` (`hasPermission`, `hasRole`, `hasOnlyRole`). Keep permission strings aligned across API and UI.
- Frontend API helpers: `$api` (`resources/js/utils/api.js`) wraps `ofetch` to append the `accessToken`, auto-handle 401/route-missing by clearing cookies + redirecting to `/login`, and centralize toast messaging; `useApi` (`resources/js/composables/useApi.js`) is the reactive fetch alternative. Prefer these utilities over ad-hoc `fetch` so headers/timeouts stay consistent.
- Frontend routing: `unplugin-vue-router` autogenerates routes from `resources/js/pages` (names kebab-cased); extra routes/redirects in `resources/js/plugins/1.router/additional-routes.js`; ACL mapping in `resources/js/plugins/1.router/routes/{hrm,crm,chat}.js` enforced by `guards.js`. When adding protected pages, also add a `canAccess` rule.

Provide `initializeEcho(token)` after login if token changes.
